# /sim - 장기 실패 시뮬레이션

> 코드 작성 후 배포 전, 아직 발생하지 않은 버그를 미리 잡습니다.

---

## S1: 실패 시뮬레이션

변경된 파일을 실제로 Read한 뒤, "3개월 후 + 규모 10배" 상황에서 코드를 따라가며 터지는 지점을 찾습니다.

```
[S1] 실패 시뮬레이션

[준비] 변경된 파일 + 의존 파일 실제 Read (추측 금지)

[상황 설정] "3개월 후, 사용자/데이터 10배"
  - 이 함수에 요청이 들어온다
  - 이 변수가 이 값이 된다
  - 여기서 이 조건이 안 맞아서 터진다

[추적 대상]
  - 하드코딩된 값 (확장자, 모델명, 경로, 인코딩)
  - 의존성 변화 (API 버전, 라이브러리, 파일 형식)
  - 규모 한계 (메모리, 처리 시간, 동시 접속)
  - 환경 차이 (OS, 인코딩, 권한)
```

규칙:
- 반드시 실제 코드를 Read한 후 시뮬레이션 시작
- "터질 수 있다" 금지 → "이 파일 N번째 줄에서 이 입력이면 터진다" (구체적)
- 코드를 따라가는 과정을 보여줘야 함

---

## S2: 근본 원인 탐색

```
[S2] 근본 원인

[근본 원인 탐색 루프]
  원인 후보 발견 →
  "이걸 고치려면 다른 걸 먼저 고쳐야 하는가?"
    → Yes: 아직 근본 원인 아님. 그 하위 레이어 코드/설정 Read 후 계속
    → No:  여기가 수정 위치. 탐색 종료

[같은 유형 전체 탐색]
  - 발견한 근본 원인과 같은 패턴이 코드 어디에 더 있나?

[CoVe 검증]
  Step 1: "이 문제가 실제로 발생하는가?" 검증 질문 생성
  Step 2: 발견 내용을 보지 않고 독립적으로 답변
  Step 3: 검증 통과한 항목만 보고

[출력]
  | # | 터진 위치 | 근본 원인 | 같은 유형 N건 | 심각도 |
```

규칙:
- 표면 원인에서 멈추지 말 것 → 최하위 레이어까지 탐색
- "아마도" 금지 → 확인된 것만 서술

---

## S3: 수정 + 확인

```
[S3] 수정 + 확인

[수정 위치 강제 검증] 코드 작성 전 필수
  S2 근본 원인 위치: {파일:라인 또는 레이어}
  S3 수정 위치:     {파일:라인 또는 레이어}
  → 불일치 시 수정 금지, S2 재탐색
  → fallback 패턴(if X is None: return default_X) 금지
     근본 원인 위치 직접 수정만 허용

[수정]
  - S2 근본 원인 위치를 직접 수정
  - 같은 유형 전체 일괄 수정
  | # | 대상 | 변경 전 | 변경 후 | 근거 |

[확인 시뮬레이션] S1을 다시 1회 실행
  - 수정한 코드로 같은 상황 재시뮬레이션
  - regression 체크: 수정 전 동작하던 기능이 수정 후에도 동작하는지 확인

[결과 분기]
  → 통과: 종료 → KI에 패턴 저장
  → 실패: 사용자에게 보고 + 승인 후 디버깅 전환
```

규칙:
- 루프 최대 2회
- 2회 후에도 실패 시 사용자 승인 필요

---

## 출력 형식

```
================================================================
[/sim 결과]
================================================================

[S1] 읽은 파일 N개 / 터진 위치: {파일:라인}

[S2] 근본 원인
  | # | 위치 | 원인 | 같은 유형 | 심각도 |

[S3] 수정
  | # | 대상 | 변경 전 | 변경 후 |

[확인 시뮬레이션] 통과 / 실패

[KI 저장] {새 패턴 있으면 기재 / 없으면 생략}

[종합] OK / WARN / FAIL
================================================================
```
