# EZ Harness - Antigravity Rules

> ez-harness Antigravity 버전. Claude Code 버전과 동일한 핵심 규칙을 Antigravity 구조에 맞게 적용.

---

## 절대 규칙 (최우선)

1. 이모지 금지 - 텍스트로 대체 ([OK], [FAIL], [Step])
2. 한글 응답
3. 보안 필수 - API 키 환경변수만, 하드코딩 금지
4. 코드 생성 전 GATE 실행 필수
5. 3개 이상 파일 변경 시 토론 없이 코드 작성 금지
6. 파일을 읽지 않고 기억으로 코드 언급 금지

---

## [역추적] 버그 수정 전 근본 원인 확정

트리거: `에러, 오류, 안되, 안됨, 문제, 실패, 버그, 왜, 500, 404, 터짐, crash, fail`

```
[역추적]
증상: {에러/문제 1줄}
경로: {어떤 입력 → 어디를 거쳐 → 여기서 터짐}
수정 위치: {근본 원인 위치 확정}
→ 수정 위치 확정 후 코드 작성
```

규칙:
- 수정 위치 확정 전 코드 작성 금지
- 증상 위치와 근본 원인 위치가 다를 수 있음 → 반드시 추적

---

## [GATE] 코드 전 정보 로드

코드 생성 전 1회 실행. 세션 내 반복 불필요.

```
[GATE 통과]
- 버전: {사용 중인 주요 라이브러리 버전 확인}
- 관련 경고: {같은 유형 실수 이력 있으면 출력}

코드 생성을 시작합니다.
```

KI(Knowledge Items)에 이전 세션 교훈이 있으면 로드 후 관련 경고 출력.

---

## [DMAD 토론] 코드 작성 전 설계 검증

3개 이상 파일 변경 시 코드 전 필수. 토론 없이 코드 작성 금지.

**토론 전 강제**: 변경 대상 파일 + 직접 의존 파일 전부 Read 완료 후 시작.

```
[DMAD 토론]

[설계자 - 순방향]
Q1. 더 단순한 방법이 있나?
Q2. 기존 코드 어디와 부딪히나?
Q3. 가장 위험한 가정 하나는?
Q4. 기존 코드베이스에 중복 로직?

[사용자 - 역방향]
Q1. 처음 보는 사람이 이 기능을 찾고 쓸 수 있나?
Q2. 기대하는 방식으로 동작하나?
Q3. 실패 시 스스로 해결할 수 있나?
Q4. 에러가 조용히 삼켜지는 곳은 없나?

규칙:
- 전부 수긍 금지, 매 라운드 반례 1개 이상
- 문제 제기 시 코드 라인 직접 인용: 파일명:라인: 코드내용
- 인용 없는 주장 금지

[토론 결과]
설계자 제안: {설계안}
사용자 반박: {반례}
최종 합의: {수정안}
```

**스킵 허용 조건** (반드시 사유 출력):
- 동일 패턴 반복 적용 (리네이밍, 포맷팅)
- 설정/배포/단순 리팩토링 (신규 기능 없는 변경)
- 사용자가 "토론 스킵" 명시 요청

## [sim] 장기 실패 시뮬레이션

코드 작성 후 3개 이상 파일 변경 시 자동 실행.
/sim 명령으로 수동 실행 가능.

상세 절차: `.agents/workflows/sim.md` 참조

---

## [smartLoop] 실패 시 최소 지점 재시작

문제 해결될 때까지 최대 3회 반복.
전체 재시작 금지 - 실패 유형별 최소 지점에서 재시작.

| 실패 유형 | 재시작 지점 |
|----------|------------|
| lint 오류 | 직접 수정만 |
| 빌드 실패 | 역추적만 재실행 |
| sim FAIL | sim S2부터 |
| 설계 결함 | 전체 재시작 |

3회 후에도 실패 시 사용자에게 보고 후 승인 대기.

## [POST-BUILD] 코드 생성 후 확인

```
[검증]
[A] API 키 하드코딩: [OK/FAIL]
[B] import 누락: [OK/FAIL]
[C] 같은 유형 버그 다른 파일에도 존재: [OK/FAIL]
결과: {N}개 문제 / 통과
```

## KI 활용 (장기 기억)

- 세션 시작 시 관련 KI 스캔 후 경고 출력
- sim에서 발견한 패턴 → 새 KI로 저장
- 같은 유형 실수 3회 이상 → KI severity 상향
- KI 형식:
  ```
  제목: {패턴명}
  증상: {어떤 상황에서}
  원인: {왜 발생}
  해결: {어떻게 고침}
  재발 방지: {다음에 어떻게 체크}
  ```
